# Admin Dashboard User Manual Guide

## Introduction

The Admin Dashboard is the private area of the website used to manage public content and review website activity.

It is used to:

- update page text and website information
- create and manage posts or announcements
- upload images and files
- review submitted testimonials
- update homepage, About page, admissions, contact details, and other public website sections

This dashboard is intended for:

- non-technical staff
- first-time users
- office administrators
- school administrators

This guide is written for users with little or no technical background.

## Getting Started

### How to Access the Admin Dashboard

1. Open the school website in your browser.
2. Go to the admin login page or use the admin access link provided by your school.
3. If you are not signed in, the login form will appear automatically.

Note:
The exact website address may be different depending on your school setup.

### How to Log In

1. Enter your staff email address.
2. Enter your password.
3. Click **Sign in to dashboard**.
4. Wait for the dashboard to load.

If your login does not work:

1. Check your email address for typing mistakes.
2. Check your password carefully.
3. Try again.
4. If the problem continues, contact your system administrator.

### Overview of the Dashboard Layout

The dashboard has three main parts:

- **Sidebar menu**:
  Used to move between sections such as Dashboard, Posts, Testimonials, and Site Management.
- **Main workspace**:
  The area where you read, edit, upload, review, and save content.
- **Action buttons**:
  Buttons such as **Save changes as draft**, **Publish changes to website**, **Create new post**, and **Reload list**.

Common labels you will see:

- **Draft**:
  Saved, but not yet visible on the public website.
- **Published**:
  Visible on the public website.
- **Pending review**:
  Waiting for approval.
- **Featured**:
  Marked to appear in a highlighted area, such as the homepage.

## How to Use Each Feature (Step-by-Step)

## Website Content / Settings

In this system, website settings are managed in the **Site Management** section.

### Edit Text and Website Information

1. Open **Site Management** from the sidebar.
2. Select the page you want to update.
3. Click inside the field you want to change.
4. Type the new text.
5. If needed, use the upload button to add a new image or file.
6. If the section has a visibility switch, turn it on or off.
7. Click **Save changes as draft** to keep your work without making it public yet.
8. Click **Publish changes to website** when you want the public to see the changes.

### Save Changes as Draft

Use this when:

- your work is not finished
- someone needs to review the changes first
- you want to continue editing later

Steps:

1. Make your changes.
2. Click **Save changes as draft**.
3. Wait for the confirmation message.

### Publish Changes to Website

Use this when:

- the content is final
- the public should see the update now

Steps:

1. Make sure your edits are correct.
2. Optional: enter an **Update summary**.
3. Click **Publish changes to website**.
4. Confirm the action.

### Reset Draft Changes

Use this when you want to remove unpublished edits.

Steps:

1. Click **Reset draft to last published version**.
2. Confirm the reset.
3. The draft will return to the last published version.

### Homepage

Use this page to update:

- the main homepage banner
- homepage buttons
- trust highlights
- homepage statistic cards
- extra homepage sections

To update homepage text:

1. Open **Site Management**.
2. Select **Homepage**.
3. Edit the **Main banner headline** and **Main banner supporting text**.
4. Update button text if needed.
5. Save as draft or publish.

To update homepage statistic cards:

1. Scroll to **Homepage statistic cards**.
2. Click **Add stat card** to add a new one.
3. Enter the **Stat label**.
4. Enter the **Number to display**.
5. Turn **Show on website** on or off.
6. Save your changes.

To update trust highlights:

1. Scroll to **Homepage trust highlights**.
2. Click **Add trust highlight** if needed.
3. Enter the **Trust highlight text**.
4. Save your changes.

### About Page

Use this page to update:

- introduction text
- mission statement
- vision statement
- principal's message
- school timeline
- leadership, faculty, and staff profiles

To add a timeline entry:

1. Open **About Page** in Site Management.
2. Find **School timeline**.
3. Click **Add timeline entry**.
4. Enter the **Year or date**.
5. Enter the **Timeline description**.
6. Turn **Show on website** on or off.
7. Save your changes.

To add or edit a staff member:

1. Go to **Leadership, Faculty & Staff**.
2. Enter the **Staff member name**.
3. Enter the **Position or department**.
4. Upload a staff photo if needed.
5. Optional: enter a **Display order**.
6. Click **Add staff member** or **Save staff changes**.

To remove a staff member:

1. Find the staff profile in the list.
2. Click **Delete profile**.
3. Confirm the action.

## Announcements / Posts

Note:
In this system, posts are used for announcements, news, and similar updates.

### Create a Post

1. Open **Posts** from the sidebar.
2. Click **Create new post**.
3. Enter the **Post title**.
4. Choose a **Post category** if needed.
5. Select the **Publishing status**:
   1. Choose **Draft** if the post should not be public yet.
   2. Choose **Published** if the post should appear on the website.
6. If needed, turn on **Show this post in featured areas on the homepage**.
7. Upload one or more images.
8. Enter the **Main post content**.
9. Click **Save post**.

### Edit a Post

1. Open **Posts**.
2. Use the search box or filters if needed.
3. Find the post you want to edit.
4. Click the edit button.
5. Update the content.
6. Click **Save post**.

### Publish or Return a Post to Draft

1. Open the post.
2. Go to **Publishing status**.
3. Choose:
   1. **Published** to make it public
   2. **Draft** to remove it from public view
4. Click **Save post**.

### Delete a Post

1. Open **Posts**.
2. Find the post.
3. Click the delete button.
4. Confirm the action.

### Feature a Post on the Homepage

1. Open the post.
2. Turn on **Show this post in featured areas on the homepage**.
3. Save the post.

### Use Filters in the Posts List

1. Open **Posts**.
2. Use the search field to search by post title.
3. Use the status filter to view only drafts or published posts.
4. Use the content filter to narrow the list further.

### Use Bulk Actions

1. Open **Posts**.
2. Select one or more posts using the checkboxes.
3. Choose one of the available actions:
   1. **Delete selected posts**
   2. **Publish selected posts**
   3. **Move selected posts to draft**

## Media Library

Note:
The current system does not have a separate **Media Library** page.

Images and files are uploaded inside the area where they are used.

### Upload Images for Posts

1. Open a post.
2. Go to **Visibility and images**.
3. Click **Select images to upload**.
4. Choose one or more images from your computer.
5. Save the post.

### Remove Images from a Post

1. Open the post.
2. Find the uploaded image preview.
3. Click the remove button on the image.
4. Save the post.

### Replace an Image in Site Management

1. Open the page section where the image is used.
2. Click the upload button for that image area.
3. Select the new image.
4. Save your changes.
5. Publish when ready.

### Upload Downloadable Files

1. Open **Admissions Page** in Site Management.
2. Go to **Downloadable admission forms**.
3. Add a new download or edit an existing one.
4. Click **Upload downloadable file**.
5. Save your changes.

Note:
There is no central page for browsing every uploaded file in one place.

## Best Practices

- Use **Draft** when content still needs checking.
- Publish only after you review the page carefully.
- Keep headings short and clear.
- Use complete, easy-to-read sentences for public text.
- Double-check spelling before publishing.
- Use clear button text, such as **Start Enrollment** or **Contact Admissions**.
- Check links before saving.
- Update one section at a time when making large changes.
- Review the public website after publishing important changes.

### When to Use Draft vs Publish

- Use **Draft** for unfinished work or content that still needs approval.
- Use **Publish** only when the content is final and ready for the public.

### Naming and Formatting Tips

- Use clear post titles such as **Enrollment Schedule for School Year 2026-2027**.
- For page link names, use lowercase letters, numbers, and hyphens only.
- Keep homepage highlights and card labels short.
- Use full website links when a field asks for a link.

## Troubleshooting

### I Cannot Log In

1. Check that the email address is correct.
2. Check that the password is correct.
3. Try signing in again.
4. If the problem continues, contact your system administrator.

### My Changes Are Not Showing on the Website

1. Check whether you only saved a draft.
2. If needed, click **Publish changes to website**.
3. Refresh the public page.
4. If the problem continues, sign out and sign in again.

### An Image or File Did Not Upload

1. Check your internet connection.
2. Try again.
3. Use a smaller file if the file is very large.
4. If the problem continues, contact your system administrator.

### I Saved a Post but Cannot See It Publicly

1. Open the post again.
2. Check the **Publishing status**.
3. Make sure it is set to **Published**.
4. Save the post again.

### I Was Signed Out Automatically

1. Sign in again.
2. Save work often to avoid losing changes.

Note:
The current system appears to sign out inactive users automatically for security.

## Important Notes

- Deleting posts or testimonials may be permanent.
- Always read the confirmation message before deleting something.
- Saving a draft does not make the content public.
- Publishing makes the content visible on the website.
- Some user accounts may have fewer permissions than others.

Note:
Project documentation suggests that some roles may be allowed to save drafts but not publish changes.

## Glossary

### Draft

A saved version that is not yet visible on the public website.

### Publish

To make content visible on the public website.

### Media

Images or files used in posts or website sections.

### Link

A website address that opens another page or file.

### Featured

Content marked to appear in an important or highlighted part of the website, such as the homepage.

### Category

A label used to group similar posts together.
